This section allows you to directly expose tags via the WebSockets API, for use in places like the integratedFreeBoard Dashboard creator.

Empty read perms disable exposing the tag, unless it is exposed via code. Any values set here override those in code, blank values default to the one set in code. These settings control access to the tag in the FreeBoard dashboard creator

Permissions are a comma separated list, The user must have all of them

Alerting

Setting blank name deletes that alarm. Conditions are python expressions that can use the tag itself(tag), the value(value), and the kaithem object. Conditions are regular subscribers and can poll at the tag's interval.

You cannot delete hardcoded alarms, but you can disable by setting the condition to False or 0

You can also use tv('name') to get any numeric tag value by name inside your expressions, stv('name') to get string tags, and np to access numpy. You also have math, random, re, and dateutil.parser.

Kaithem tracks the tags you use in this way, and the alarms will re-evaluate when needed.

Loggers

These loggers control the logging of tag data to history.db
Interval is the maximum rate to log at. No logs are created if there are no changes. Tags that change more often than the interval will always 'flush' the most recent value after sufficient time.
Accumulate determines what happens to multiple readings within an interval. To store the latest, use latest. To store the mean for the whole interval, use average.
Set the interval to 0 to delete the logger. Set history length to 0 to disable deleting old data.
Set the target to ram to only log in memory, but beware filling availible space.
